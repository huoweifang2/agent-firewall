# Agent-Firewall × OpenClaw 中文说明书

## 1. 项目定位

本项目只保留一个主产品方向：**我自己的 Telegram bot 入口 + Agent-Firewall 中间层 + 本机 OpenClaw runtime**。

核心原则：

- Telegram 是唯一默认对话入口。
- `localhost:3000` 是后台控制台，不是主要聊天入口。
- 输入扫描、工具调用、OpenClaw skill、MCP 调用都必须经过 Agent-Firewall。
- 所有关键事件都写入 audit log 和 agent trace。
- 默认本地运行使用 `uv`、SQLite、memory cache，不依赖 Docker。

## 2. 主链路

```text
Telegram Bot
  -> Telegram Bridge
  -> Proxy /v1/scan
  -> Agent runtime graph
  -> pre-tool gate
  -> OpenClaw skill / MCP provider
  -> post-tool gate
  -> trace + audit log
  -> Telegram reply
```

当 Proxy 判定输入为 `BLOCK`，或 pre-tool gate 判定需要人工确认时：

1. Telegram 立即回复用户：请求已暂停，等待控制台审批。
2. Proxy 创建 `/v1/interventions` 记录。
3. 前端 **Approvals / Audit** 显示待审批项。
4. 审批通过后，Agent worker 继续执行并回发 Telegram。

## 3. 本地配置位置

- `~/.openclaw/openclaw.json`：OpenClaw 主配置、DeepSeek provider、Telegram bot token。
- `~/.openclaw/agent-firewall.json`：Agent-Firewall 接管 Telegram 的 bridge 配置。
- `~/.openclaw/agent-firewall.sqlite`：本地 SQLite 数据库。
- `~/.openclaw/agent-firewall-plugin-runtime-deps`：Agent-Firewall 调 OpenClaw 时使用的可写插件依赖目录。
- `apps/proxy-service/.env.local`：Proxy 本地配置，禁止提交。
- `apps/agent/.env.local`：Agent/Telegram Bridge 本地配置，禁止提交。

当前默认只接管 `agent-firewall.json` 中配置的 `default` Telegram account；其他 Telegram account 不会自动启用。

## 4. 本地启动

```bash
make setup
./start-local.sh
```

访问：

- 前端：`http://localhost:3000`
- Proxy：`http://localhost:8000`
- Agent：`http://localhost:8002`

默认配置：

```env
DATABASE_URL=sqlite+aiosqlite:////Users/isaachuo/.openclaw/agent-firewall.sqlite
CACHE_BACKEND=memory
REDIS_URL=
ENABLE_LANGFUSE=false
TELEGRAM_BRIDGE_ENABLED=true
OPENCLAW_AGENT_ID=coder
```

## 5. 页面分工

- **Attack Playground**：默认首页、左侧第一项，用于提示词攻击和拦截演示。
- **Approvals / Audit**：处理 Telegram 拦截和工具确认。
- **Skills & Hooks**：发现 OpenClaw skills/hooks，把 skill 绑定为受保护工具。
- **OpenClaw Sandbox**：本地调试 OpenClaw runtime、工具流和 trace。
- **Trace / Audit**：审计每轮输入、工具计划、工具门控、工具输出和最终回复。
- **Runtime Settings**：查看脱敏后的 DeepSeek、OpenClaw、Telegram Bridge、gateway 状态。

## 6. 联调检查

```bash
curl http://127.0.0.1:8000/health
curl http://127.0.0.1:8002/health
curl http://127.0.0.1:8002/agent/openclaw/config
```

普通对话：

```bash
curl -X POST http://127.0.0.1:8002/agent/chat \
  -H 'Content-Type: application/json' \
  -d '{"message":"hello","user_role":"customer","session_id":"manual-smoke"}'
```

审批队列：

```bash
curl 'http://127.0.0.1:8000/v1/interventions?status=pending'
```

## 7. 不再作为主路径的内容

- Docker Compose、Postgres、Redis、Langfuse 不是默认本地路径。
- reference-chat-target 和旧电商 demo 不再作为产品服务。
- root benchmark 文档不再作为当前项目入口；当前重点是 Telegram bot runtime、防火墙审批和完整日志。
