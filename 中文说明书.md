# Agent-Firewall × OpenClaw 中文说明书

## 1. 项目定位

本项目现在收敛为一个 **OpenClaw 外壳 + Agent-Firewall 安全网关**：

- Telegram 是优先入口：用户消息先进入 Agent-Firewall Telegram Bridge。
- Agent-Firewall 先调用 Proxy `/v1/scan` 做输入拦截。
- 允许通过后，再调用本机 OpenClaw agent。
- 前端不是主要聊天入口，而是控制台、调试台和毕业展示界面。
- 模型主路径统一为 DeepSeek 官方 API；浏览器 API key 仅作为 legacy override。

主链路：

```text
Telegram Bot
  -> apps/agent Telegram Bridge
  -> apps/proxy-service /v1/scan
  -> apps/agent /agent/chat
  -> openclaw agent --agent coder
  -> Telegram reply
```

## 2. 页面分工

- Playground：DeepSeek 一问一答，以及提示词攻击防御演示。
- Compare：展示 protected OpenClaw 与 direct OpenClaw 的差异。
- Security Scan：保持现状，作为安全扫描和批量验证页面。
- OpenClaw Sandbox：OpenClaw agent 外壳，展示 runtime gates、trace、tool flow。
- Middleware：OpenClaw skills、hooks、MCP 的发现与绑定页。
- Runtime Settings：只展示 DeepSeek、OpenClaw、Telegram、gateway、本地配置状态。

## 3. 本地配置

主要配置位置：

- `/Users/isaachuo/.openclaw/openclaw.json`：OpenClaw 主配置、DeepSeek provider、Telegram bot 原始配置。
- `/Users/isaachuo/.openclaw/agent-firewall.json`：Agent-Firewall 接管 Telegram 的 bridge 配置。
- `/Users/isaachuo/.openclaw/agent-firewall.sqlite`：Agent-Firewall 本地 SQLite 数据库。
- `/Users/isaachuo/.openclaw/agent-firewall-plugin-runtime-deps`：Agent-Firewall 调 OpenClaw 时使用的可写插件运行依赖目录。
- `/Users/isaachuo/Agent-Firewall/apps/proxy-service/.env.local`：Proxy 本地 DeepSeek、OpenClaw 配置。
- `/Users/isaachuo/Agent-Firewall/apps/agent/.env.local`：Agent 服务本地 DeepSeek、OpenClaw、Telegram Bridge 配置。

注意：

- 不要把真实 API key、Telegram token、gateway token 写入仓库。
- 不要在 UI 或 API 响应中回显 key/token。
- OpenClaw 原生 Telegram 已关闭，由 Agent-Firewall Telegram Bridge 接管，避免同一个 bot 被两个消费者同时轮询。

## 4. Docker 是否需要

默认本地方案已经不需要 Docker：

- 数据库：SQLite，文件为 `/Users/isaachuo/.openclaw/agent-firewall.sqlite`。
- 缓存：进程内 memory cache，不再要求 Redis。
- Langfuse：默认关闭，不影响 `/health`、`/v1/scan`、`/agent/chat`、Telegram Bridge 或前端演示。
- Docker Compose：保留为可选 legacy/fallback，不作为本地默认路径。

Proxy 默认配置：

```env
DATABASE_URL=sqlite+aiosqlite:////Users/isaachuo/.openclaw/agent-firewall.sqlite
CACHE_BACKEND=memory
REDIS_URL=
ENABLE_LANGFUSE=false
```

如需全新初始化，可先删除 SQLite 文件；启动 Proxy 时会自动建表并 seed 默认 policies、rules、OpenClaw Gateway agent、tools、roles、skills：

```bash
rm -f /Users/isaachuo/.openclaw/agent-firewall.sqlite*
```

## 5. 本地启动顺序

推荐一键启动，在项目根目录：

```bash
/Users/isaachuo/Agent-Firewall/start-local.sh
```

手动启动时，先设置无 Docker 默认环境：

```bash
export DATABASE_URL=sqlite+aiosqlite:////Users/isaachuo/.openclaw/agent-firewall.sqlite
export CACHE_BACKEND=memory
export REDIS_URL=
export ENABLE_LANGFUSE=false
```

启动 Proxy：

```bash
cd /Users/isaachuo/Agent-Firewall/apps/proxy-service
uv run python -m uvicorn src.main:app --host 127.0.0.1 --port 8000
```

启动 Agent：

```bash
cd /Users/isaachuo/Agent-Firewall/apps/agent
uv run --extra dev python -m uvicorn src.main:app --host 127.0.0.1 --port 8002
```

启动前端：

```bash
cd /Users/isaachuo/Agent-Firewall/apps/frontend
npm run dev
```

访问：

- 前端：http://localhost:3000
- Proxy：http://localhost:8000
- Agent：http://localhost:8002

## 7. 当前已处理的关键点

- Agent 主路径已从旧 graph runtime 改为 OpenClaw-only。
- Proxy 安全扫描 pipeline 已移除 LangGraph 依赖，改为本地顺序 runner。
- Proxy 默认运行已改为 SQLite + memory cache + Langfuse disabled，无需 Docker。
- DeepSeek 成为唯一主模型 provider。
- Telegram Bridge 已接入 Agent-Firewall，再转 OpenClaw。
- OpenClaw 插件运行依赖目录改为可写的 Agent-Firewall 专用目录，绕开 root-owned `plugin-runtime-deps`。
- OpenClaw model status 进入 UI/API 前会做递归脱敏。

## 8. 常见问题

如果 OpenClaw 报 `EACCES`：

- 优先确认 Agent-Firewall 是否使用了 `OPENCLAW_PLUGIN_STAGE_DIR=/Users/isaachuo/.openclaw/agent-firewall-plugin-runtime-deps`。
- 如需修复 OpenClaw 原始目录，可手动运行：

```bash
sudo chown -R "$(id -un):staff" /Users/isaachuo/.openclaw/plugin-runtime-deps
```

如果 Telegram 没回复：

- 检查 OpenClaw 原生 Telegram 是否关闭，Agent-Firewall bridge 是否开启。
- 检查 Telegram bot 是否已被 webhook 占用。
- 检查 `.openclaw/agent-firewall/telegram-offsets.json` 是否记录了过大的 offset。
- 重启 Agent 服务后再发新消息，不要复用旧 pending update 判断结果。
